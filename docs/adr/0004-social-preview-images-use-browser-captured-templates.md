# Social preview images use browser-captured templates

Rocket creates Default Social Preview Images from author-facing Social Preview Templates rendered through browser-based Social Preview Capture, because normal HTML and CSS give Site Authors the styling control they expect. SVG templates were rejected because authoring precise layout was too painful, and native image composition libraries were rejected because they add difficult platform-specific build dependencies. To keep browser capture practical, Rocket fingerprints pixel-affecting inputs, reuses unchanged generated images, supports static build-time delivery everywhere, and lets deployment Adapters opt into request-time generation and caching. Social Preview Template Preview stays development-time by default so authors can tune template HTML without creating public preview Pages.
