# Standalone Demo URLs use generated child paths

Standalone Demo URLs are public, stable URLs in development and deployed builds. Rocket uses generated child paths under the parent Markdown Page, with the reserved `_demo` segment, the exact demo export name, and a canonical trailing slash, so a Page like `/basics/demos/` can keep its default static output while `/basics/demos/_demo/simpleButton/` can be generated as a separate Standalone Demo document. Standalone Demo URLs are generated automatically for every JavaScript Demo in a Markdown Page because the JavaScript Demo frame exposes Standalone Demo URL behavior by default. They are directly addressable generated documents, not configured Pages, so they do not appear in Page registries, menus, or PageData navigation trees. Standalone Demo URLs inherit the parent Markdown Page's rendering mode: static Markdown Pages generate physical Standalone Demo documents, while server-rendered Markdown Pages route the same URL shape through the adapter and Page Runtime. Parent Markdown Pages keep the JavaScript Demo frame with source and Standalone Demo URL controls, while Standalone Demo documents render only the live demo without frame, source, controls, navigation chrome, or a built-in back link so the demo can use the full page. Unknown demo names return 404. If a configured Page collides with a generated Standalone Demo URL, the build fails instead of choosing precedence. Query-string and hash variants were rejected because static hosts do not reliably route them to distinct generated documents, and Rocket does not keep query-string Standalone Demo URLs as a compatibility path.
