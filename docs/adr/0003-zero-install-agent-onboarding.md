# Zero-install agent onboarding starts with public docs

Rocket's primary first-start path for Site Authors is Zero-install Agent Onboarding: a public `/setup/build-with-ai` docs page with a self-contained starter prompt and generated project-local Rocket Agent Skill, not a prerequisite installed Rocket-specific skill or plugin. The Rocket home page and Start section should point Site Authors toward building with a Coding Agent first, while keeping `/setup/manual-quick-start` available as the fallback path for authors who want to build by hand; the home page should expose only those two setup calls to action, `Start with AI` and `Manual Quick Start`, instead of also linking to reference material. This keeps the first experience zero-install and agent-agnostic, while still allowing Rocket skills, plugins, or richer tool integrations to be added later as optional power-user paths. After `@rocket/js` is installed, the Rocket Initializer may create a removable project-local Rocket Agent Skill because the Site Author has explicitly chosen Rocket for the project; that skill is a maintenance aid, not a prerequisite for first start. The guaranteed outcome is a Deployable Rocket Site, and publishing to Netlify is an optional branch when the Site Author gives the Coding Agent hosting access.
